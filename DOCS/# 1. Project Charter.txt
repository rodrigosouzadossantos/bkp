# 1. Project Charter

## 1.1 Project Title

**Subsea Benthic Analysis AI – Multi-Modal Environmental Intelligence Pipeline**

---

## 1.2 Purpose

The project aims to develop an Artificial Intelligence pipeline for the automated analysis of subsea datasets, enabling identification and filtering of benthic-relevant environmental content to support environmental baseline studies.

---

## 1.3 Objectives

### Primary Objective (MVP)

Develop a **binary classification system** capable of distinguishing:

* Data containing **benthic-relevant features**
* Data not containing benthic-relevant features (including anthropic or irrelevant content)

---

### Secondary Objectives

* Establish a scalable multi-modal data processing pipeline
* Enable AI-assisted annotation workflows to reduce manual effort
* Prepare architectural foundations for future:

  * Object detection
  * Segmentation
  * Taxonomic classification

---

## 1.4 Definition of Benthic-Relevant Data (MVP Operational Definition)

Data is considered **benthic-relevant** if it contains:

* Visible **seafloor substrate**
* **Benthic organisms** (flora/fauna associated with the seabed)
* **Ecological structures**:

  * Reefs
  * Sediment patterns
  * Bioturbation features

### Explicit Exclusions

* Open water column (no visible seabed)
* Purely anthropogenic structures **without biological presence**
* Unusable data:

  * Severe turbidity
  * Motion blur
  * Extreme lighting degradation

---

## 1.5 MVP Strategy

### **“Filter Noise, Preserve Confidence”**

The MVP prioritizes **precision over recall**, meaning:

* It is **acceptable to retain anthropic or irrelevant content**
* It is **not acceptable to incorrectly discard benthic-relevant data with high confidence**

**Strategic Goal:**
Deliver a **high-confidence filtered dataset**, even if partially conservative, to support expert workflows.

---

## 1.6 Success Criteria

### Model Performance

* Accuracy ≥ 60% (balanced evaluation subset)
* Precision ≥ 75% for benthic-relevant class (**primary KPI**)
* Controlled false negatives (monitored, not eliminated)

---

### Operational Impact

* Dataset reduction ≥ 40–60%
* Annotation time reduction ≥ 40%
* ≥ 85–90% retention of true benthic-relevant data

---

### Validation

* Usability validated by marine domain experts
* Outputs considered reliable for downstream annotation

---

## 1.7 Scope

### Included

* Multi-modal data ingestion:

  * Images (primary for MVP)
  * Video (frame extraction only)
  * Sonar (future compatibility)
  * Bathymetry (future compatibility)
* Data preprocessing and normalization (basic level)
* MVP model development (binary classification – still imagery)
* AI-assisted annotation workflow design
* Human-in-the-loop validation framework
* Preparation for future phases

---

### Excluded

* AUV operations and data acquisition
* Real-time or streaming deployment
* Full taxonomic classification (e.g., WoRMS integration)
* Advanced multimodal fusion (deferred)
* Anthropogenic object analysis beyond filtering

---

## 1.8 High-Level Requirements

### Functional Requirements

* Binary classification of benthic relevance
* Batch processing of large datasets
* AI-assisted annotation support
* Human validation interface capability

---

### Non-Functional Requirements

* Scalability to TB-scale datasets
* Offline-first processing capability
* Explainability (traceable model outputs)
* Logging and auditability
* Dataset versioning support

---

### Compliance Requirements

* Use of **license-compliant AI frameworks only**
* Clear traceability of model/data usage

---

## 1.9 Constraints

* No pre-labeled dataset available
* Natural **class imbalance (non-benthic dominance)**
* High environmental variability:

  * Turbidity
  * Lighting attenuation
  * Backscatter
* Multi-modal data complexity
* Licensing restrictions
* Offline-first processing requirement
